What drives the progress data?

How does the dashboard know what's done and what's not?

A

Manual checkboxes in the UI

You click to mark phases/tasks as done directly in the dashboard. Progress is saved to the database (or localStorage). Simple, no file parsing needed.

B

Reads from specs/main/tasks.md

The dashboard parses your actual SpecKit task file. Tasks marked [x] count as done. Truth lives in the file — dashboard just visualizes it.

C

Hardcoded phases + manual toggle

The 11 build phases (0–10) are hardcoded from CLAUDE.md. You toggle each phase done/not done. No file parsing, no DB — just a config you update as you build.
